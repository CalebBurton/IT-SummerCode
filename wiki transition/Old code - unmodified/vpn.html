Gathering Information

TNS-DataNetwork needs all of the following information in order to troubleshoot VPN issues. Collect all of the following details before referring any VPN issues to them or another support group.

  • How are you connecting to VPN? (Which OS are you using? Are you using your operating system's native VPN or Cisco AnyConnect?)
  • Have you successfully connected to VPN before? If so, were you on the same network as you are now, or were you somewhere else?
  • What kind of WiFi network are you on now – home? work? hotel? public? Are you connecting from another country, and if so which country?
  • Do you receive a specific error message when you try to connect? If so, could you send us a screenshot or provide the full text of the error?
  • Please provide the external and internal IP addresses of your device by following the instructions at https://kb.northwestern.edu/page.php?id=69163
  • Please provide the date and time you last tried to connect to the VPN.

Troubleshooting

If a user contacts us with any issue connecting to the VPN, follow these steps:

  1. If the user hasn't configured VPN yet, help them download and use the Cisco VPN client; see the appropriate configuration article for instructions. If they already have Cisco AnyConnect on their device, make sure the program is running and that the user has tried connecting to the VPN in the program. If they are using Native VPN or another VPN client and aren't in Feinberg, help the user install and use Cisco AnyConnect instead.
  2. If Cisco doesn't work or the user can't use Cisco (ex. if they are a Feinberg user), help the user configure Native VPN/remove and reconfigure existing connections that don't work.
  3. If the native VPN produces an error, see if there are any known solutions to the error by searching for it in the KB. If the error's not listed in the KB, see if there are any known solutions on the web.
  4. If you cannot resolve the error, collect all the information under Gathering Information in General  of this page and refer the user to departmental support (if connecting on a University-owned device) or TNS-DataNetwork (for personally-owned devices).

Referrals

  • All unresolvable issues with VPN configuration on university-owned machines should go to department support for further assistance. Tier 1 should always try to set up the connection (using Bomgar if needed) before routing elsewhere.
  • Support escalations from IT staff should go to DataNetwork after collecting information under Gathering Information in General of this page.
  • If a student or staff/faculty member with personal devices needs configuration support and has trouble with our provided instructions, we can do the configration for them at Laptop ER.
  • All other unresolvable isues should go to NUIT-CI-TNS-DataNetwork after collecting all the information under General.
